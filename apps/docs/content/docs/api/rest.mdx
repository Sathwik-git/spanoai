The SDK is a wrapper over this HTTP + WebSocket API. You can call it directly from any
language.

## Conventions

- **Base URL** — `http://localhost:8000` in dev.
- **Auth** — either an API key (`X-SpanoAI-Key: spanoai_sk_…`, for agents/SDK/MCP) or a
  dashboard session token (`Authorization: Bearer …`, for a logged-in user) on every
  request except `/health` and `/auth/signup` · `/auth/login`.
- **Identity** — `X-SpanoAI-Agent: <agentId>` names the calling agent.
- **Bodies** — JSON in, JSON out. Errors are `{ error, message, requestId }` with the
  appropriate HTTP status.

## Accounts & keys

A user signs up (provisioning a tenant), logs in for a session token, then mints API
keys for their agents. Key management is gated to the session token — an API key cannot
create more keys.

## WebSocket stream

Real-time events (new messages, context changes) are pushed over a WebSocket.

<Steps>

<Step>
### Mint a ticket
```bash
curl -X POST http://localhost:8000/stream-ticket \
  -H "X-SpanoAI-Key: $KEY" -H "X-SpanoAI-Agent: planner" \
  -H "Content-Type: application/json" -d '{ "sessionId": "run-1" }'
# → { "ticket": "…", "expiresIn": 30 }   (single-use, ~30s TTL)
```
</Step>

<Step>
### Connect
```text
ws://localhost:8000/stream/run-1?ticket=<ticket>
```
The stream supports gap recovery — on reconnect you can resume from the last event id
so nothing is missed across a blip.
</Step>

</Steps>
